Prompts: Gerador de Hipóteses de Produto

Prompts para levantar hipóteses de produto, explorar oportunidades e estruturar experimentos com IA.

Como usar esses prompts

Use quando tiver dados de usuários — entrevistas, analytics ou feedbacks — e quiser transformá-los em hipóteses estruturadas para priorizar e testar com o time.

Lembre: hipótese boa é testável e falsificável. "Os usuários querem uma experiência melhor" não é hipótese. "Se reduzirmos o onboarding de 8 para 4 passos, a taxa de conclusão aumentará em 20%" é uma hipótese válida.

Prompt 1 — Gerar hipóteses a partir de dados

🤖 Gerador de hipóteses
Sou PM de [produto] e tenho os seguintes dados: Dados quantitativos: [ex: 60% dos usuários abandonam no passo 3 do onboarding] Dados qualitativos: [ex: 5 usuários mencionaram confusão com o campo de configuração inicial] Contexto: [produto, segmento e perfil do usuário] Gere 5 a 8 hipóteses no formato: "Se [intervenção], então [resultado esperado], porque [justificativa baseada nos dados]." Para cada hipótese, indique: - Nível de confiança: alto, médio ou baixo — com base nos dados fornecidos - Como eu testaria essa hipótese (experimento, entrevista, A/B test, etc.) - O que precisaria medir para validar ou refutar

Prompt 2 — Explorar oportunidades de produto

🤖 Exploração de oportunidades
Produto: [descrição do produto] Segmento de usuário: [quem usa] Principal métrica que queremos mover: [ex: retenção em 30 dias] Dados disponíveis: [o que você sabe sobre o comportamento atual] Ajude-me a explorar oportunidades respondendo: 1. Quais jobs to be done esse usuário tem que ainda não estamos resolvendo bem? 2. Onde os usuários provavelmente têm mais fricção hoje? 3. Quais comportamentos dos usuários de maior sucesso poderíamos tentar replicar? 4. Que feature poderia reduzir o esforço do usuário pela metade? 5. Que problema adjacente nosso produto não resolve e que esse usuário provavelmente tem?

Prompt 3 — Estruturar um experimento

🤖 Estruturar experimento
Quero testar a seguinte hipótese: [descreva sua hipótese] Contexto: - Volume de usuários: [ex: 5.000 usuários ativos por mês] - Métrica principal que vou medir: [descreva] - Restrições: [tempo disponível, esforço de engenharia, riscos] Ajude-me a estruturar um experimento que: 1. Defina claramente o grupo de controle e o grupo de teste 2. Especifique qual mudança mínima será feita para testar a hipótese 3. Estime o tamanho de amostra necessário para um resultado estatisticamente significativo 4. Defina critérios claros de sucesso e de fracasso antes de começar 5. Identifique os principais riscos e como mitigá-los

Prompt 4 — Análise de concorrentes para discovery

🤖 Análise de concorrentes
Estou fazendo discovery para um produto de [categoria]. Concorrentes conhecidos: [liste os principais] Problema que estamos investigando: [descrição] Segmento de usuário: [quem] Com base no seu conhecimento sobre esse mercado: 1. Quais são as principais abordagens que os concorrentes usam para resolver esse problema? 2. Que lacunas ou pontos fracos são comuns nessa categoria de produto? 3. Quais são as reclamações mais frequentes de usuários de produtos similares? 4. Que diferenciação seria mais difícil de copiar nesse mercado? 5. Que tendências do mercado devo considerar no meu discovery? Use como ponto de partida para pesquisa — não como verdade absoluta.
